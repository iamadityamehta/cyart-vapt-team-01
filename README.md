# 🔐 Security Assessment and Vulnerability Analysis Using Open-Source Tools

## 📌 Introduction
This project focuses on performing a comprehensive security assessment and vulnerability analysis using open-source tools. The goal is to identify potential vulnerabilities in a system, analyze their impact, and provide remediation strategies to enhance overall security posture.

---

## 📌 Internship Context

This task was completed as a part of my Vulnerability Assessment and Penetration Testing (VAPT) Internship at Cyart Technologies in March 2026.

---

## 🎯 Objective
- To understand the fundamentals of security assessment
- To identify vulnerabilities using open-source tools
- To analyze risk severity and potential impact
- To suggest effective remediation techniques

---

## 📍 Scope
- Vulnerability scanning using OpenVAS
- Basic system/network security assessment
- Identification of misconfigurations and weak points
- Documentation of findings and recommendations

---

## 🛠️ Tools Used
- OpenVAS (Greenbone Vulnerability Manager)
- Kali Linux / Linux Environment
- Networking utilities (if used)

---

## ⚙️ Methodology
1. **Setup Environment**
   - Installed and configured OpenVAS
   - Ensured services are running properly

2. **Target Identification**
   - Selected system or network for scanning

3. **Vulnerability Scanning**
   - Performed scan using OpenVAS
   - Collected scan results

4. **Analysis**
   - Categorized vulnerabilities (Low, Medium, High, Critical)
   - Studied CVEs and their impact

5. **Reporting**
   - Documented vulnerabilities
   - Suggested remediation steps

---

## 📊 Key Findings
- Identified multiple vulnerabilities across different severity levels
- Found outdated software and misconfigurations
- Detected potential entry points for attackers

---

## 💡 Key Learnings
- Understanding of vulnerability assessment process
- Hands-on experience with OpenVAS
- Importance of system hardening and updates
- Risk prioritization based on severity
- Real-world exposure to cybersecurity practices

---

## ⚠️ Challenges Faced
- Difficulty in setting up OpenVAS environment
- Errors during scanning process
- Performance issues during scan execution
- Understanding complex vulnerability reports

---

## 🛡️ Remediation Strategies
- Regular system updates and patch management
- Closing unused ports and services
- Strengthening configurations
- Applying security best practices
- Continuous monitoring and scanning

---

## 📌 Conclusion
This project provided practical exposure to security assessment and vulnerability analysis. It helped in understanding how attackers identify weaknesses and how organizations can proactively secure their systems using open-source tools.
